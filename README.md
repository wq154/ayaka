# sr-apocalyptic-plugin（ayaka）

> TRSS-Yunzai 插件 · 崩坏：星穹铁道「混沌回忆 / 虚构叙事 / 末日幻影」动态三队战绩 + 「异相仲裁 / 王棋 / 王琪」战绩查询。

这是一个给 **TRSS-Yunzai** 使用的星铁战绩查询插件。它复用 `miao-plugin` 已有的 UID、Cookie、米游社鉴权与 Puppeteer 渲染能力，专门补齐星铁高难战绩里“三队/多节点”展示不完整的问题。

## 功能概览

- **混沌回忆 / 忘却之庭**：支持 `node_1 ~ node_n` 动态识别，三队关卡不再被固定成两队。
- **虚构叙事**：按关卡节点动态渲染队伍、积分、星数与周期信息。
- **末日幻影**：支持三队节点、积分汇总与本期/上期查询。
- **异相仲裁 / 王棋 / 王琪**：保留独立查询入口，统一紫金科技风图片模板。
- **@群友 / 显式 UID 查询**：优先识别消息中的 UID；没有 UID 时复用 Yunzai 绑定 UID 流程。
- **验证码兜底**：深渊类接口会尝试复用 ji-plugin 的验证码处理链路。

## 特性

- 优先级 `-999999`，用于优先接管星铁高难战绩类命令，避免旧模板只显示两队。
- 动态识别 `node_1 ~ node_n`，当前三队以及未来更多节点都能兼容。
- 混沌 / 虚构 / 末日 / 王棋按钮入口统一，方便连续切换查询。
- 图片渲染失败时自动降级为文字结果。
- 发布版已脱敏：仓库不包含个人 UID 原始返回、调试 raw 数据、备份文件；运行时默认不保存米游社原始响应。

## 环境要求

- TRSS-Yunzai
- 已安装并配置可用的 `ji-plugin` / `miao-plugin`
- 已绑定星铁 UID
- 需要可用的米游社 Cookie / CK（使用现有扫码登录、刷新 CK 流程即可）

## 安装

在 TRSS-Yunzai 根目录执行：

```bash
cd plugins
git clone https://github.com/wq154/ayaka.git sr-apocalyptic-plugin
cd ..
pnpm install
```

或直接将本仓库放入：

```txt
TRSS-Yunzai/plugins/sr-apocalyptic-plugin
```

然后重启 TRSS-Yunzai。

## 更新

```bash
cd plugins/sr-apocalyptic-plugin
git pull
```

重启 TRSS-Yunzai 后生效。

## 隐私与脱敏说明

- 本仓库不提交 `data/`、`data/raw/`、`.bak`、运行日志等本地调试/缓存文件。
- 插件只通过现有 Yunzai / ji-plugin 流程读取 UID 与 CK，不内置任何账号 Cookie、Token 或私有配置。
- 发布版已移除原始接口响应落盘逻辑，避免把个人 UID、角色昵称、战绩原始 JSON 写入插件目录。
